# 医疗决策 AI 顾问 · 完整产品方案

> 创业原则：先服务愿意付费且高频使用的陪诊师，再反哺患者端；先闭环 1 家医院决策数据，再扩张医院网络。  
> 传播靠一页纸报告，护城河靠闭环数据，现金流靠 B 端提效。

---

## 一、产品定位

### 一句话定位

面向复杂就医决策的**个人医疗 AI 顾问**——先帮你想清楚「现在该做什么」，再匹配医生能力与就医路径，而不是替你抢一个热门号。

### 核心价值

| 价值 | 说明 |
|------|------|
| 降低信息不对称 | 科室 / 医生 / 决策阶段错配 |
| 降低白跑成本 | 少一次无效挂号与奔波 |
| 降低错误决策 | 把「名气」换成「能力匹配」 |
| 提升陪诊交付 | 标准化报告与路径可复用，抬客单 |

### 与普通挂号平台的区别

| 维度 | 挂号平台 | 本产品 |
|------|----------|--------|
| 卖什么 | 号源可达 | 决策质量 |
| 排序逻辑 | 职称 / 热度 | 决策阶段 × 能力画像 |
| 结束点 | 挂号成功 | 路径、费用预期、随访闭环 |
| 数据 | 点击与成交 | 建议→修正→结果 |

### 与通用 AI 问答的区别

| 维度 | 通用 AI | 本产品 |
|------|---------|--------|
| 产出 | 知识 | 可执行决策与路径 |
| 约束 | 弱绑定医院 | 绑定湘雅等真实流程 |
| 沉淀 | 难成训练资产 | 强制闭环案例 |
| 合规 | 易越界诊断 | 边界写进交互与话术 |

### 用户为什么会主动传播

1. **决策报告一页纸**（真问题 / 阶段 / 下一步 / 医生能力）——适合发家庭群求共识  
2. **陪诊师专业规划 PDF**——抬形象与客单，天然获客物料  
3. **「少白跑一次」故事**——痛感强、结果可感知  
4. **脱敏决策复盘**——比科普更可执行，易被二次引用  

**设计原因**：传播物必须是「报告」而不是 App 截图——报告可独立于产品使用。

## 二、用户核心场景（≥5）

| # | 场景 | 真实焦虑 | 决策阶段 | 产品交付 | 传播点 |
|---|------|----------|----------|----------|--------|
| 1 | 我该挂哪个科 | 挂错科白跑、被转诊 | 风险确认 / 初诊 | 科室匹配理由 + 首诊材料清单 | 「原来不是随便挂某科」 |
| 2 | 我该选哪个医生 | 名气 ≠ 当前阶段需要 | 方案选择 / 手术决策 | 能力画像 Top3 + 证据与不确定项 | 「终于有人说证据不足」 |
| 3 | 是否要去北京/上海 | 外地成本 vs 增量确定性 | 方案选择 | 本地湘雅可解决度 + 转诊触发条件 | 「不一定非要北上」 |
| 4 | 不想只吃药（如心理） | 被长期开药、缺非药物路径 | 长期管理 | 流派/治疗方式能力匹配（如 CBT） | 「按诉求匹配，不按职称」 |
| 5 | 想做微创是否适合 | 创伤、恢复、适应证不确定 | 手术决策 | 决策清单 + 需向医生确认的问题 | 「带着问题去问诊」 |
| 6 | 检查单看不懂 | 害怕耽误、过度解读 | 风险确认 | 报告翻译（非诊断）+ 复诊窗口 | 「焦虑被安放」 |
| 7 | 陪诊师 10 分钟出规划 | 形象与效率、客单难涨 | 全阶段 | 一键报告 / 报价依据 | 客户觉得「很专业」 |

### 第一阶段优先级

- **P0**：选医生能力匹配、就医路径规划（湘雅）  
- **P1**：科室匹配、报告翻译  
- **P2**：是否北上、微创决策清单  
- 陪诊师侧所有 P0 **必须可导出报告**

## 三、应用信息架构

### 患者端（C）

```
首页
 ├─ AI 医疗决策（主路径）
 ├─ 医生匹配
 ├─ 就医规划
 ├─ 我的病例
 └─ 个人健康档案
```

### 陪诊 / 运营（B）

```
陪诊工作台（产能中心）
 ├─ 案例库
 ├─ 医生能力图谱
 └─ 运营数据看板
```

**为什么这样设计**：早期日活不在患者逛 App，而在陪诊师把工具当「工作记忆」；患者端保持极简入口。

### 角色权限

| 模块 | 患者 | 陪诊师 | 运营 |
|------|------|--------|------|
| AI 决策对话 | 读写自己 | 读写客户单 | 只读抽检 |
| 医生匹配 | 查看 | 编辑修正 | 维护能力标签 |
| 就医规划报告 | 查看/分享 | 生成/导出/报价 | 模板配置 |
| 案例库 | 公开脱敏精选 | 全文检索复用 | 审核入库 |
| 健康档案 | 本人 | 授权可见 | 脱敏统计 |

## 四、关键页面设计

### 4.1 首页

| 项 | 内容 |
|----|------|
| 目标 | 3 秒进入决策，而不是逛内容 |
| 模块 | 主输入框、高频场景、最近报告、合规提示条 |
| 流程 | 输入困扰或点场景 → 进入 AI 决策 |
| AI 角色 | 意图粗分类（科/医生/路径/报告） |
| 哇塞瞬间 | 一句话就被分到「决策阶段」，而不是丢百科 |

### 4.2 AI 医疗决策

| 项 | 内容 |
|----|------|
| 目标 | 完成四层：真问题 → 阶段 → 约束 → 下一步 |
| 模块 | 对话区、阶段标签、约束卡片、下一步行动清单 |
| 流程 | 5–8 轮结构化问答 → 生成决策摘要 |
| AI 角色 | 需求理解 + 阶段判断 + 风险提示协作 |
| 哇塞瞬间 | 说出没说出口的担忧：「你怕的是耽误，不是结节本身」 |

### 4.3 医生匹配

| 项 | 内容 |
|----|------|
| 目标 | 按能力画像给 Top3，证据可点开 |
| 模块 | 需求能力列表、医生卡、匹配度、证据/不确定 |
| 流程 | 确认能力需求 → 查看匹配 → 收藏/导出 |
| AI 角色 | 医生能力匹配 Agent |
| 哇塞瞬间 | 主动写「证据不足」，比盲目推荐更可信 |

### 4.4 就医规划

| 项 | 内容 |
|----|------|
| 目标 | 把决策变成可执行行程与材料包 |
| 模块 | 日程、材料、科室顺序、外地清单、费用区间 |
| 流程 | 确认约束 → 生成规划 → 分享家属/陪诊 |
| AI 角色 | 路径规划 Agent |
| 哇塞瞬间 | 一张图讲清「这次来长沙怎么少跑」 |

### 4.5 我的病例 / 个人健康档案

| 项 | 内容 |
|----|------|
| 目标 | 轻量沉淀检查单、对话、历史报告 |
| 模块 | 上传、时间线、授权给陪诊师 |
| AI 角色 | 结构化抽取（谨慎） |
| 哇塞瞬间 | 下次复诊不用从头讲故事 |

### 4.6 陪诊工作台

| 项 | 内容 |
|----|------|
| 目标 | 10 分钟出专业报告，抬客单 |
| 模块 | 客户列表、AI 草稿、修正面板、导出、结果回填 |
| 流程 | 建单 → AI → 改 → 发客户 → 回填挂号/结果 |
| AI 角色 | 全链路草稿；人工是质量闸门 |
| 哇塞瞬间 | 客户说「比我自己挂号靠谱」；陪诊师愿为提效付费 |

### 4.7 案例库

| 项 | 内容 |
|----|------|
| 目标 | 沉淀可复用决策，形成训练语料 |
| 模块 | 检索、标签、前后对比、随访字段 |
| 流程 | 结案强制回填 → 审核入库 |
| AI 角色 | 结构化抽取 + 脱敏 |
| 哇塞瞬间 | 同类案例「上次怎么走的」一眼可见 |

## 五、AI Agent 体系设计

### 编排原则

编排层只做路由与合并。不让单一大模型「又诊断又推荐又规划」。冲突时优先 **风险提示** 与 **证据不足**。

### Agent 一览

| Agent | 输入 | 输出 | 职责边界（禁区） |
|-------|------|------|------------------|
| 需求理解 | 用户口述/表单 | 真问题、隐性担忧、约束标签 | 不判断病情轻重；不给诊断定论 |
| 决策阶段 | 真问题+病史摘要+检查列表 | 五阶段之一 + 置信度 | 不跳阶段直接推专家；低置信则多问 |
| 医生能力匹配 | 阶段+能力需求+医院范围 | Top3 + 证据引用 | 不按名气排序；无证据标不足 |
| 就医路径规划 | 约束+匹配结果+院内流程知识 | 日程/材料/路径/奔波优化 | 不承诺约到某医生号源 |
| 报告翻译 | 检查报告文本/OCR | 通俗解释+建议确认项 | 不做阴阳性最终判定；不开药 |
| 风险提示 | 全链路中间态 | 红旗提醒、合规话术改写 | 不吓人、不包治；急症只敦促急诊 |
| 陪诊报告生成 | 合并结果+人工修正 | 客户可读 PDF/飞书文档 | 不伪装医院官方意见 |

### 推荐调用顺序

```
需求理解 → 决策阶段 →（并行）报告翻译 / 风险提示
        → 医生能力匹配 → 路径规划 → 陪诊报告生成
人工修正节点：匹配之后、导出之前
```

## 六、数据资产设计（最重要）

### 核心闭环字段

| 数据对象 | 关键字段 | 谁产生 | 长期价值 |
|----------|----------|--------|----------|
| 患者画像 | 地域、预算、奔波/沟通偏好、决策风格 | 问答+陪诊 | 个性化路径先验 |
| 初始诉求 | 原话、场景标签、隐性担忧 | 患者/陪诊 | 意图分布与迭代 |
| AI 建议 | 阶段、下一步、匹配医生、证据快照 | Agent | 可回放模型输出 |
| 陪诊师修正 | 改了什么、为什么改 | 陪诊师 | **最贵的专家知识蒸馏** |
| 最终挂号医生 | 院区/科室/医生/号源渠道 | 回填 | 匹配准确率真值 |
| 医生实际建议 | 保守/手术/继续观察等结构化 | 回填 | 能力画像校准 |
| 患者最终选择 | 遵从/改道/放弃及原因 | 回填 | 依从与错配分析 |
| 费用区间 | 预估 vs 实际 | 回填 | 规划可信度 |
| 满意度 | 决策帮助度、是否少白跑 | 问卷 | 付费转化信号 |
| 30 天结果 | 复诊、方案调整、主观改善 | 随访 | 短期有效性 |
| 长期随访 | 90/180 天节点 | 随访 | 慢病与手术结局资产 |
| 医生能力标签 | 技术/病种/流程友好/证据来源 | 运营+案例反哺 | 可扩展匹配引擎 |

### 为什么这是长期护城河

1. **修正数据** = 廉价标注的高价值专家知识  
2. **结果数据**让匹配从「像真的」变成「被验证过」  
3. 案例越多，陪诊师越离不开工具（工作记忆在系统里）  
4. 医院扩张时可迁移的是 **决策本体**，不是单一院内黄页  

> 竞品可抄 UI，抄不走你的修正率与 30 天结局分布。

## 七、商业模式

| 模式 | 怎么收费 | 阶段适配 | 风险 |
|------|----------|----------|------|
| 患者付费 | 单次报告 / 月度顾问会员 | 中期 | 获客贵、信任慢 |
| **陪诊师付费** | SaaS 席位 + 报告额度 | **第一阶段首选** | 需证明提效与涨客单 |
| 服务分成 | 陪诊/绿通订单抽成 | 有交易闭环后 | 依赖供给履约 |
| 高净值医疗规划 | 家医式年费 | 品牌起来后 | 交付重、难规模化 |
| 保险/企业合作 | 按人次嵌入福利 | 有案例与合规后 | 销售周期长 |

### 第一阶段现金流结论

**陪诊师 SaaS + 报告增值**。

原因：ROI 清晰（省时间、涨客单、可传播 PDF）；比患者更愿为专业工具付费；同时强制回填，反哺模型。患者端前期可免费获客，付费点放在「完整报告导出 / 家属共享版」。

### 定价锚点（可执行）

| 档位 | 价格 | 权益 |
|------|------|------|
| 陪诊师 Starter | ¥99–199/月 | 30 份报告 · 基础匹配 |
| 陪诊师 Pro | ¥399–699/月 | 无限草稿 · 案例库 · 团队 |
| 患者单次 | ¥29–99/次 | 完整决策报告（可先免） |

## 八、合规边界

### 能做

- 就医目标梳理与决策阶段识别  
- 基于公开信息的医生能力方向匹配  
- 路径、材料、奔波与费用区间提示  
- 报告通俗化翻译（非诊断结论）  
- 红旗症状时建议及时急诊 / 线下就医  

### 不能做

- 诊断疾病、判断轻重、承诺疗效  
- 推荐具体药品与剂量  
- 替代医嘱或「必须找某医生」  
- 伪造号源 / 内部渠道承诺  
- 超必要采集并滥用敏感数据  

### 避免被认定为「医疗诊断」的产品策略

| 策略 | 落地方式 |
|------|----------|
| 角色锚定 | 每页固定：决策辅助，不替代医生 |
| 输出句式 | 「更适合关注的能力方向」≠「你应确诊为…」 |
| 证据分层 | 公开证据 / 陪诊经验 / 证据不足 |
| 人机闸门 | 陪诊导出前强制「已人工审阅」 |
| 急症分流 | 红旗词只给急诊指引，关闭匹配 |
| 日志可审计 | 提示词版本、引用快照、人工修正差 |

### 医生推荐合规表达示例

> 基于公开资料，该医生长期开展××方向，与你当前「方案选择阶段」的能力需求较匹配；是否适合仍需线下评估。不构成诊疗建议。

## 九、MVP（30 天版本）

### 硬约束

- 7 天 Demo 可演示  
- 30 天有真实陪诊师闭环  
- 不做原生 App、不做复杂挂号对接、不做自动爬取医生站  
- 可用：**扣子 + 飞书 + Cursor**

### 规模数字

| 项 | 数量 |
|----|------|
| 页面 | **6** |
| 数据表 | **8** |
| 扣子工作流 | **5**（可合并为 3） |
| 医院范围 | **1（湘雅）** |
| 医生库 | 先 30，月内到 80–150（人工维护） |

### 功能清单

| 必须有 | 明确不做 |
|--------|----------|
| 结构化问答（决策四层） | 自动抢号 / HIS 对接 |
| 湘雅医生能力库（人工） | 全网医生自动挖掘 |
| 匹配 Top3 + 证据字段 | 疗效承诺与病情判断 |
| 就医规划一页纸 + 飞书/PDF | 原生 App 双端 |
| 陪诊修正与结果回填 | 复杂权限中台 |
| 案例脱敏入库（手动审核） | 保险理赔系统 |

### 6 个页面

1. 落地页/入口  
2. 决策对话  
3. 匹配结果  
4. 规划报告  
5. 陪诊工作台（含回填）  
6. 案例列表（可用飞书多维表代替）

### 8 张数据表（飞书多维表即可）

`users` · `cases` · `dialogues` · `ai_outputs` · `companion_edits` · `doctor_profiles` · `bookings_feedback` · `followups`

### 5 个工作流（扣子）

1. 需求理解  
2. 阶段判断  
3. 医生匹配  
4. 路径规划报告  
5. 对话脱敏入库（可选）

### 每天 / 每周开发计划

| 时间 | 焦点 | 验收标准 |
|------|------|----------|
| D1–D2 | 需求冻结、话术边界、医生表字段 | 字段字典 + 合规文案定稿 |
| D3–D7 | Cursor Web Demo：问答→结果；录入 30 位医生 | 可对外演示主路径 |
| D8–D14 | 扣子接通；飞书表；陪诊修正表单 | 1 名陪诊师能独立出报告 |
| D15–D21 | 医生库→100；报告模板；强制回填 | 10 个真实案例闭环 |
| D22–D30 | 打磨匹配；访谈 5 名陪诊；定价试验 | ≥3 人愿付费或明确意向；修正率可统计 |

### 验证指标（比 DAU 重要）

报告完成率 · 人工修正率 · 回填率 · 「是否少白跑」主观分 · 付费意向

## 十、长期演化路线

| 阶段 | 形态 | 关键动作 | 为何此时做 |
|------|------|----------|------------|
| 1. 湘雅陪诊工具 | B 端工作台 + 薄 C 端 | 闭环数据、付费陪诊师、能力库 | 供给集中、场景深、现金流近 |
| 2. 区域医院决策网络 | 多院能力图谱 + 转诊规则 | 复制湘雅二/三及其他三甲 | 路径从单院变成网络 |
| 3. 个人医疗 AI 顾问 | 长期档案与主动随访 | 会员制、家庭账户、跨次记忆 | 有足够结局数据才配「顾问」 |
| 4. 全球医疗决策平台 | 跨境就医决策网络 | 多语言、多体系、合规本地化 | 国内方法论与数据本体可迁移 |
